## 一、基于物理的相机（Physical Camera）

### 1.1 概述

传统游戏相机使用抽象的 FOV（视场角）和手动调节的曝光值，与真实摄影没有对应关系。基于物理的相机使用真实摄影参数——焦距、光圈、快门速度、ISO、传感器尺寸——来自动推导出渲染所需的 FOV、景深、运动模糊和曝光补偿。

### 1.2 核心参数

| 参数 | 符号 | 单位 | 典型值 | 说明 |
|------|------|------|--------|------|
| 焦距 | \(f\) | mm | 24-200 | 广角 < 35mm < 标准 ~50mm < 长焦 |
| 光圈 | \(N\) (f-stop) | 无量纲 | 1.4-22 | f/1.4(大光圈,浅景深) → f/22(小光圈,深景深) |
| 快门速度 | \(t\) | 秒 | 1/8000-30 | 1/1000s(冻结运动) → 1/30s(运动模糊) |
| ISO 感光度 | \(S\) | 无量纲 | 50-25600 | ISO 100(低噪点) → ISO 6400(高噪点,弱光) |
| 传感器宽度 | \(W_s\) | mm | 36(全画幅) | 36×24 全画幅, 23.6×15.6 APS-C, 17.3×13.0 M4/3 |
| 传感器高度 | \(H_s\) | mm | 24(全画幅) | 用于计算垂直 FOV |
| 对焦距离 | \(d_f\) | 世界单位 | 0.1-∞ | 相机到清晰对焦平面的距离 |

### 1.3 焦距 → FOV 推导

传感器尺寸和焦距共同决定视场角：

\[
\text{fov}_{\text{vertical}} = 2 \times \arctan\left( \frac{H_s}{2 \times f} \right)
\]

**推导**：传感器半高 \(H_s/2\) 与焦距 \(f\) 形成直角三角形，半 FOV 为其反正切值。

**示例**：
- 全画幅 36×24mm + 50mm 镜头 → FOV ≈ 27°（标准镜）
- 全画幅 36×24mm + 24mm 镜头 → FOV ≈ 53°（广角）
- 全画幅 36×24mm + 200mm 镜头 → FOV ≈ 6.9°（长焦）
- APS-C 23.6×15.6mm + 50mm 镜头 → FOV ≈ 17.7°（等效全画幅 75mm）

**与当前引擎对比**：硬编码 FOV = 60°，等同于全画幅 20mm 超广角镜头。

### 1.4 光圈孔径

\[
D = \frac{f}{N}
\]

- 孔径直径 \(D\) 以 mm 为单位
- \(f/2.8\), 50mm 镜头 → 孔径 = 17.86mm（大光圈，浅景深）
- \(f/16\), 50mm 镜头 → 孔径 = 3.13mm（小光圈，深景深）

### 1.5 曝光三角形（光圈 × 快门 × ISO）

曝光值 EV (Exposure Value) 定义为：

\[
\text{EV} = \log_2\left( \frac{N^2}{t} \times \frac{100}{S} \right)
\]

**物理意义**：
- 光圈开大一档（N 减小 √2 倍）→ EV +1 → 光量翻倍
- 快门速度减半（t 减半）→ EV +1 → 光量翻倍
- ISO 翻倍 → EV -1 → 感光度翻倍

**曝光偏置**：以 "f/2.8, 1/60s, ISO100" 为基准点（EV≈8.88），计算当前参数与基准的 EV 差值：

\[
\Delta\text{EV} = \text{EV}_{\text{ref}} - \text{EV}_{\text{current}}
\]

\(\Delta\text{EV} > 0\) 表示需要更多曝光（当前参数相对基准曝光不足），叠加到 AutoExposure 输出：

\[
E_{\text{final}} = E_{\text{auto}} \times 2^{\Delta\text{EV}}
\]

**示例**：
- 基准 f/2.8, 1/60s, ISO100 → ΔEV = 0（不变）
- f/5.6, 1/60s, ISO100 → ΔEV = +2（暗两档，曝光×4）
- f/2.8, 1/125s, ISO100 → ΔEV = +1（暗一档，曝光×2）
- f/2.8, 1/60s, ISO400 → ΔEV = -2（亮两档，曝光×0.25）

### 1.6 景深（Depth of Field）弥散圆

**薄透镜模型（简化版）**：

\[
\text{CoC}_{\text{world}} = D \times \frac{|d_f - d|}{d}
\]

其中 D 为光圈孔径直径，\(d_f\) 为对焦距离，\(d\) 为像素处场景深度。

归一化到屏幕空间：

\[
\text{CoC}_{\text{screen}} = \frac{\text{CoC}_{\text{world}}}{H_s}
\]

钳制范围：0.005 ~ 0.05（0.5% ~ 5% 屏幕宽度）。

**物理直觉**：
- 大光圈（f/1.4, 孔径大）→ 大 CoC → 浅景深（人像摄影）
- 小光圈（f/16, 孔径小）→ 小 CoC → 深景深（风光摄影）
- 长焦镜头（焦距大）→ 同等 f-stop 下孔径大 → 浅景深

### 1.7 运动模糊强度

\[
I_{mb} = \frac{t}{t_f}
\]

其中 \(t\) 为快门速度，\(t_f\) 为帧间隔。

- 1/60s 快门 @ 60fps → \(I_{mb} = 1.0\)（正常 1 帧曝光）
- 1/30s 快门 @ 60fps → \(I_{mb} = 2.0\)（2 帧曝光，更模糊）
- 1/1000s 快门 @ 60fps → \(I_{mb} = 0.06\)（近乎无模糊，冻结运动）

### 1.8 完整推导流程

```
PhysicalCameraParams
  │  focalLength, fStop, shutterSpeed, iso, sensor, focusDistance
  │
  ├─→ FocalLengthToFOV(focalLength, sensorHeight) ──→ fov ──→ 投影矩阵
  ├─→ ApertureDiameter(focalLength, fStop) ──→ apertureDiameter
  │     └─→ MaxCoC(apertureDiameter, sensorHeight) ──→ maxCoC ──→ DOF Pass
  ├─→ ExposureBias(fStop, shutterSpeed, iso) ──→ exposureBias ──→ ToneMap Pass
  └─→ MotionBlurIntensity(shutterSpeed, frameTime) ──→ mbIntensity ──→ MotionBlur Pass
```

## 二、基于物理的光源（Physical Lights）

### 2.1 概述

传统游戏光源使用无单位 "intensity" 乘数 + 美术向衰减曲线。基于物理的光源使用真实光度学单位——流明、坎德拉、勒克斯——配合物理正确的平方反比衰减和色温，与物理相机的曝光系统协同工作，形成完整的物理光照管线。

### 2.2 光度学单位体系

| 物理量 | 符号 | 单位 | 说明 |
|--------|------|------|------|
| 光通量 | \(\Phi\) | 流明 lm | 光源向所有方向辐射的总可见光能量 |
| 发光强度 | \(I\) | 坎德拉 cd = lm/sr | 单位立体角内的光通量 |
| 照度 | \(E\) | 勒克斯 lux = lm/m² | 单位面积上接收的光通量 |
| 亮度 | \(L\) | 尼特 cd/m² | 单位面积在给定方向上的发光强度 |

**关系链**：

```
流明 lm ──(除以立体角 sr)──→ 坎德拉 cd ──(除以距离² m²)──→ 勒克斯 lux
```

点光源：\(I = \frac{\Phi}{4\pi}\)（全向均匀辐射）

照度：\(E = \frac{I}{d^2}\)（平方反比定律）

### 2.3 光源类型与物理模型

#### 2.3.1 点光源 — 发光强度（坎德拉 cd）

点光源的物理量是**发光强度** \(I\)（单位 cd = lm/sr）。

\[
E(d) = \frac{I}{d^2} \quad [\text{lux}]
\]

**现实参考值**：
- 蜡烛：~1 cd
- 60W 白炽灯泡：~70 cd（~880 lm 全向）
- 100W LED 灯泡：~120 cd（~1500 lm 全向）
- 汽车前大灯远光：~10,000 cd
- 灯塔：~10⁶ cd

**着色器实现**：

```
radiance = light.color × light.luminousIntensity × (1.0 / max(d², 0.001))
```

平方反比产生极近处极亮的物理正确行为，搭配物理相机曝光后自动适配。

#### 2.3.2 聚光灯 — 发光强度（坎德拉 cd）

聚光灯物理量与点光源相同（坎德拉），额外叠加锥角衰减：

\[
E(d, \theta) = \frac{I}{d^2} \times f_{\text{cone}}(\theta) \quad [\text{lux}]
\]

其中锥角衰减使用 smoothstep 在内外锥角之间过渡。

**典型场景**：手电筒 ~500 cd，演播室聚光灯 ~10,000 cd。

#### 2.3.3 方向光 — 照度（勒克斯 lux）

方向光的物理量是**照度** \(E\)（单位 lux），因为它是从无限远的平行光源接收到的单位面积光通量。

物理上不存在真正的 "无限远" 光源，但我们用照度来近似太阳/天空光：

\[
\text{radiance} = \text{light.color} \times E \quad [\text{lux}]
\]

无距离衰减（方向光假设无限远）。

**现实参考值**：
- 正午直射阳光：~100,000 lux
- 白天阴天：~10,000 lux
- 室内办公室：~500 lux
- 黄昏：~10 lux
- 满月：~0.1 lux

**示例**：场景中设置 100,000 lux 的方向光 + 物理相机 f/2.8, 1/60s, ISO100 → 曝光输出与真实晴天的亮度级别一致。

### 2.4 平方反比衰减 vs 传统多项式衰减

**当前引擎的美术向衰减**：

\[
\text{attenuation} = \frac{1}{1 + 2d + d^2}
\]

其中 \(d = \text{distance} / \text{range}\)（归一化距离）。在范围边缘（\(d=1\)）衰减到 0.25，近处（\(d\to0\)）接近 1.0。曲线形状是美术调参的多项式，无物理依据。

**基于物理的平方反比衰减**：

\[
\text{attenuation} = \frac{1}{\max(d^2, \varepsilon)} \times \text{smoothstep}^2(1, 0, d)
\]

其中：
- \(d = \text{distance} / \text{range}\)（归一化距离）
- \(\varepsilon = 0.001\) 防止近处数值爆炸
- smoothstep² 在范围边缘提供软截止（避免硬截断）

| 距离 | 旧衰减 | 物理衰减 |
|------|--------|----------|
| d=0.01（很近） | ~0.98 | 10000×（极亮，物理正确） |
| d=0.1 | ~0.82 | 100× |
| d=0.5 | ~0.50 | 4× |
| d=1.0（范围边界） | 0.25 | 1×（截止前） |
| d>1.0 | (继续衰减) | 0（软截止） |

物理相机曝光系统会自动补偿近处的高亮度，产生自然的过曝效果。

### 2.5 色温 — Kelvin → RGB

光源色温使用开尔文（K）表示。在 CPU 端将色温转换为 RGB 颜色乘数，零 GPU 开销。

**Planck 黑体辐射近似（Kryzysztof 公式）**：

```
对于 1000K ≤ T ≤ 40000K:

红色分量:
  r = 1.0, 如果 T ≤ 6600K
  r = 0.0004×(T-6600)³ + 1.0, 钳制 [0,1]

绿色分量:
  g = -0.0087×T + 1.9284, 如果 T ≤ 6600K
  g = -0.00328×(T-6600)² + 1.0, 钳制 [0,1]

蓝色分量:
  b = 1.0, 如果 T ≤ 2000K
  b = -0.0213×(T-2000)² + 1.0, 如果 2000K < T ≤ 6600K, 钳制 [0,1]
  b = 1.0, 如果 T > 6600K
```

**典型色温**：

| 色温 (K) | 描述 | RGB (近似) |
|----------|------|------------|
| 1700 | 火柴火焰 | (1.00, 0.39, 0.05) |
| 2700 | 白炽灯 | (1.00, 0.64, 0.27) |
| 3200 | 卤素灯 | (1.00, 0.74, 0.42) |
| 4000 | 荧光灯 | (1.00, 0.85, 0.64) |
| 5500 | 正午日光 | (1.00, 0.98, 0.94) |
| 6500 | 阴天 / D65标准 | (0.98, 0.98, 1.00) |
| 10000 | 蓝天 | (0.80, 0.92, 1.00) |

### 2.6 光通量 ↔ 坎德拉 转换（全向点光源）

\[
I = \frac{\Phi}{4\pi} \quad [\text{cd}]
\]

其中 \(4\pi\) 是完整球面的立体角（约 12.57 sr）。此转换适用于全向均匀辐射的点光源。

**示例**：一个 1500 lm 的 LED 灯泡：
- 等效发光强度：\(I = 1500 / 4\pi \approx 119\) cd
- 在 2m 距离处照度：\(E = 119 / 2^2 \approx 30\) lux

### 2.7 方光源（面积光）— 后续规划

方光源（如矩形灯板、荧光灯管）使用**亮度** \(L\)（单位 cd/m²）作为物理量：

\[
E = L \times \Omega_{\text{solid}}
\]

其中 \(\Omega_{\text{solid}}\) 是光源面积对接受点所张的立体角。这需要 PDF 采样支持，属于 Phase 2 内容。

## 三、物理相机 × 物理光源的协同工作

### 3.1 完整物理光照管线

```
物理光源 (cd/lux)              PBR BRDF (GGX)              物理相机
┌──────────────────┐    ┌──────────────────────┐    ┌──────────────────┐
│ 点光: I cd       │    │ diffuse  = albedo/π  │    │ f/2.8, 1/60s     │
│ 方向光: E lux    │───→│ specular = D×G×F/... │───→│ ISO 100           │
│ 平方反比衰减     │    │ × radiance           │    │ → 曝光 EV 偏置   │
│ 色温→RGB         │    │ = 物理正确反射值     │    │ → 适配真实亮度级 │
└──────────────────┘    └──────────────────────┘    └──────────────────┘
```

### 3.2 物理一致性示例

**场景**：室内办公室
- 方向光（窗户光）：10,000 lux, 色温 5500K（日光）
- 物理相机：f/2.8, 1/60s, ISO 400

**推导**：
- 10,000 lux / (f/2.8, 1/60s, ISO400) → EV 基准偏移 +2（比基准亮两档）
- AutoExposure 自动适配亮度级别
- 曝光输出与真实室内光照条件一致

**如果切换到晚上场景**：
- 台灯点光源：50 cd, 色温 2700K（暖白）
- 距离 1m → 照度 = 50 / 1² = 50 lux
- 物理相机：f/1.4, 1/30s, ISO 3200（夜景设置）
- 曝光输出自动适配暗光环境

### 3.3 关键收益

| 方面 | 传统方案 | 物理方案 |
|------|----------|----------|
| FOV | 硬编码 60° | 焦距+传感器自动推导 |
| 曝光 | 手动调 AutoExposure 参数 | 光圈+快门+ISO 自动 |
| 景深 | 手动调 focusDepth/focusRange | 光圈+焦距+对焦距离自动 |
| 运动模糊 | 手动调 intensity=0.5 | 快门速度自动 |
| 光源强度 | 无单位 intensity × 美术衰减 | 坎德拉/勒克斯 × 平方反比 |
| 光源颜色 | 手动 RGB picker | 色温 → RGB 物理近似 |
| 近处光源 | 衰减→1.0（无过曝） | 平方反比→极高亮度（物理正确过曝） |
| 场景切换 | 重新手动调所有参数 | 保持物理参数，曝光自动适配 |
