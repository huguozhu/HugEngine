## 一、高优先级（影响物理正确性）

### 1. BRDF 多重散射补偿

**问题**：当前 Cook-Torrance GGX 单次散射模型在粗糙表面（roughness > 0.5）会大量丢失能量。微面元只模拟单次反弹，忽略光线在微面元之间的多次反弹。White Furnace 测试（全白环境 + albedo=1.0）输出显著小于 1.0。

**原理**：Kulla-Conty 近似（2017）通过在 BRDF 上叠加一个能量补偿项来修复：

\[
f_{\text{ms}}(L, V) = \frac{(1 - E(\mu_o)) \times (1 - E(\mu_i))}{\pi \times (1 - E_{\text{avg}})} \times F_{\text{avg}} \times E_{\text{avg}}
\]

其中：
- \(E(\mu)\) 是方向反照率（directional albedo），即给定方向入射光被单次散射反射的比例
- \(E_{\text{avg}}\) 是平均反照率（对半球积分）
- \(F_{\text{avg}}\) 是平均菲涅尔项

简化实现（Frostbite 拟合）：

```hlsl
// 在 PBR_BRDF() 末尾添加
float3 F_ms = F_SchlickRoughness(NdotV, F0, roughness, metallic);
float energyCompensation = 1.0 + F0 * (1.0 / directionalAlbedo(NdotV, roughness) - 1.0);
color = (diffuse + specular * energyCompensation) * NdotL;
```

**收益**：粗糙金属在明亮环境中亮度恢复 ~40-50%

**改动文件**：`pbr_common.slang`，约 40 行

### 2. 亮度校准（参考白点）

**问题**：AutoExposure 目标 0.18（18% 灰）是归一化的无单位值。没有映射到真实显示器亮度（尼特）。物理光源的坎德拉/勒克斯值无法锚定到显示输出。

**原理**：建立 SDR 参考白点：

```
SDR_REFERENCE_WHITE = 80 cd/m² (nits)   ← sRGB 显示器标准白点
18% 灰 = 80 × 0.18 = 14.4 nits

场景辐亮度 (lux) → BRDF → 传感器照度 → 曝光 → 显示亮度 (nits)
```

**实现**：

1. AutoExposure 新增 `m_DisplayWhitePoint` 参数（默认 80 nits）
2. 曝光计算改为：`exposure = whitePoint / (avgLum × targetLum)`
3. 物理相机 EV 偏置在此基础上叠加

**收益**：物理光源的 cd/lux 值与最终显示亮度建立物理联系

**改动文件**：`AutoExposurePass.h/.cpp` + `ToneMap.frag.slang`，约 50 行

### 3. 正向管线物理单位适配

**问题**：`ForwardPipeline::CollectLights` 仍使用 `LightComponent::intensity`（无单位乘数），未读取 `luminousIntensity` 和 `illuminance` 物理字段。

**实现**：镜像 DeferredPipeline 的做法——检测 `luminousIntensity > 0` 或 `illuminance > 0`，编码到 `GPULight`（`positionRange.w < 0` 标记物理模式）。

**收益**：ForwardPipeline 也支持物理光源

**改动文件**：`ForwardPipeline.cpp`，约 20 行

### 4. F0 可配置（IOR → 镜面反射率）

**问题**：当前所有电介质硬编码 F0 = 0.04（4% 垂直反射率）。无法区分不同材质：

```
水 IOR 1.33 → F0 = 0.02（2% 反射率）
玻璃 IOR 1.5 → F0 = 0.04（4% 反射率）
宝石 IOR 2.4 → F0 = 0.17（17% 反射率）
```

**原理**：Fresnel 垂直入射反射率：

\[
F_0 = \left( \frac{n_1 - n_2}{n_1 + n_2} \right)^2
\]

空气 \(n_1 = 1.0\) 时：\(F_0 = \left( \frac{\text{IOR} - 1}{\text{IOR} + 1} \right)^2\)

**实现**：
- `Material.h` 新增 `float ior = 1.5f` 字段
- `CollectLights`/GBuffer 传递 IOR 到 GPU（或 CPU 端预计算 F0）
- `pbr_common.slang` 使用 `f0` 替代硬编码 `0.04`

**收益**：支持水、宝石、玻璃等不同电介质材质的正确反射率

**改动文件**：`Material.h` + `pbr_common.slang` + `ShaderTypes.slang`，约 15 行

## 二、中优先级（扩展系统）

### 5. 物理天空/太阳模型

**问题**：当前天空是静态立方体贴图（`SkyboxPass`），无程序化大气散射。

**原理**：Hosek-Wilkie（2012）或 Bruneton（2008）大气散射模型：

```
瑞利散射（Rayleigh）：小粒子（空气分子）→ λ⁻⁴，蓝光散射更强 → 蓝天
米氏散射（Mie）：大粒子（气溶胶/水滴）→ λ⁰，波长无关 → 灰白色雾/云
```

**实现概要**：
- 新增 `PhysicalSky` 类：接受太阳方向、浑浊度、地面反照率 → 解析天空辐射度 LUT
- 替换 `SkyboxPass` 的静态贴图采样为解析式计算
- 太阳盘模型：立体角 + 基于光谱的辐照度 → 自动设置方向光 `illuminance`
- 空中透视：基于距离的大气消光应用到场景对象

**改动文件**：新增 `PhysicalSky.h/.cpp` + 修改 `SkyboxPass` + 新增 compute shader，约 300 行

---

### 6. 材质系统扩展（Disney BSDF 参数）

**问题**：当前仅支持 glTF 2.0 金属/粗糙度模型，缺少 7 个 Disney BSDF 参数。

**缺失参数**：

| 参数 | 说明 | 典型用途 |
|------|------|---------|
| `anisotropic` | 各向异性方向/强度 | 拉丝金属、毛发 |
| `subsurface` | 次表面散射 | 皮肤、蜡、大理石 |
| `specular` | 镜面反射强度（超越 F0） | 电介质自定义反射率 |
| `specularTint` | 镜面反射色调 | 彩色金属反射 |
| `sheen` | 光泽（边缘柔光） | 天鹅绒、缎子 |
| `clearcoat` | 清漆层 | 汽车漆、木材清漆 |
| `clearcoatGloss` | 清漆粗糙度 | 哑光/亮光清漆 |

**实现概要**：
- `Material.h` 新增上述字段
- `GPUMaterial`（`ShaderTypes.slang`）扩展布局
- `pbr_common.slang` 新增对应 BRDF 项

**改动文件**：`Material.h` + `ShaderTypes.slang` + `pbr_common.slang` + `GBuffer.frag.slang`，约 200 行

---

### 7. HDR 输出（HDR10 / scRGB）

**问题**：当前交换链是 `BGRA8_UNORM + sRGB`（SDR，8 位/通道）。

**HDR 格式**：

| 格式 | 传递函数 | 位深 | 峰值亮度 |
|------|---------|------|---------|
| HDR10 | ST.2084 (PQ) | 10-bit | 1000-10000 nits |
| scRGB | 线性 | 16-bit float | 无限制 |
| HLG | HLG | 10-bit | 1000 nits |

**实现概要**：
- `VulkanSwapChain` 支持 `VK_FORMAT_A2B10G10R10_UNORM_PACK32` + `VK_COLOR_SPACE_HDR10_ST2084_EXT`
- `ToneMap.frag.slang` 输出 PQ/HLG 编码替代 ACES+sRGB
- 色域映射：BT.709 → BT.2020

**改动文件**：`VulkanSwapChain.cpp` + `ToneMap.frag.slang` + `common.slang`，约 150 行
